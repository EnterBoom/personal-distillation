# Acceptance and maturity

## Ten acceptance domains

Maintain tests for:

1. identity understanding;
2. case and capability evidence;
3. offering/product understanding;
4. voice and style fidelity;
5. content or copy task performance;
6. workflow/SOP execution;
7. agent design and routing;
8. project or engineering requirements;
9. strategic decision consistency;
10. knowledge update correctness.

Each test records instruction, expected output, pass criteria, common failures, latest result, evidence, and test date. Do not mark a test passed without executing it against current knowledge.

## Maturity levels

- **V0 initialized**: structure, control docs, status, inbox rules, system records, and templates exist.
- **V1 personally usable**: control and profile are usable, at least one evidence-backed case exists, identity tests pass, and the AI avoids major identity misclassification.
- **V2 work usable**: offerings, voice, and workflows are usable; outputs meet user-defined direct-use criteria.
- **V3 agent callable**: agent, project, decision, and test modules are usable; at least 8 of 10 acceptance domains pass; purpose-specific exports are reliable.

## Test report

```markdown
# Knowledge-base acceptance report
| Test | Result | Evidence | Failure | Repair |
|---|---|---|---|---|

## Summary
- Passed:
- Current maturity:
- Blocking failures:
- Next repair:
```

Never advance maturity because files merely exist. Base it on usable/verified content and executed tests.
